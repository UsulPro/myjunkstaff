<h2><a id="user-content-major-components" class="anchor" href="#major-components" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Major Components</h2>

<p>Storybook mainly comes with few different components. Here are they:</p>

<ol>
<li>Storybook Manager UI - That's the UI of the storybook where you interact with.</li>
<li>Storybook Preview - This is the iframe your components are rendered.</li>
<li>Webpack Dev Server - Storybook runs a Webpack Dev Server with <a href="https://github.com/webpack/docs/wiki/hot-module-replacement-with-webpack">HMR</a> support. (usually, you invoke this with <code>npm run storybook</code>.)</li>
<li>Static Builder - Build the storybook into a set of static files. (usually, you invoke this with <code>npm run build-storybook</code>.)</li>
</ol>

<p>Let's have a look at each of these component with more information.</p>

Storybook Manager UI</h3>

<p>This is the management UI for Storybook where you interact with your stories and see your actions. It also loads the Storybook Preview window.</p>

<p>This is written with React according to the <a href="https://github.com/kadirahq/mantra/">Mantra</a> specification. It also use Redux as the local state. Source code for Manager lives under <code>src/client/manager</code>.</p>

<p>It has following <a href="https://kadirahq.github.io/mantra/#sec-Mantra-Modules">mantra modules</a>:</p>

<ul>
<li>api - api module, which handles different functionalities of Storybook.</li>
<li>shortcuts - handles keyboards shortcuts</li>
<li>preview - handles rendering the preview and communicate with it.</li>
<li>ui - renders the UI, routing and initializing keyboard shortcuts and so on.</li>
</ul>

<blockquote>
<p>api, shortcuts and ui modules live inside a seperate repo called at <a href="https://github.com/kadirahq/storybook-core">storybooks-core</a>. With that, we could reuse the same UI in a lot of different storybooks projects.</p>
</blockquote>

<p>We publish this manager UI as a single JavaScript file to the NPM and it doesn't depend on enduser's locally installed NPM modules. When we run <code>npm run prepublish</code> it'll build the manager UI and saved it into <code>dist/manager.js</code>.</p>

<h3><a id="user-content-2-storybook-preview" class="anchor" href="#2-storybook-preview" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>2. Storybook Preview</h3>

<p>This is the place we render components related to stories. This has a live connection to the Webpack dev server and it support <a href="https://github.com/webpack/docs/wiki/hot-module-replacement-with-webpack">HMR</a>. This usually renders as an iframe inside the Manager UI.</p>

<p>Once loaded, preview sends information like available stories, actions to the parent frame(Manager UI) via some iframe hacks. (using <a href="https://www.npmjs.com/package/page-bus">page-bus</a> module).</p>

<p>It also receives instructions from the parent window(Manager UI) to do certain tasks like switching stories.</p>

<p>Source code for Storybook preview is located at <code>src/client/preview</code>.</p>

<h3><a id="user-content-3-webpack-dev-server" class="anchor" href="#3-webpack-dev-server" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>3. Webpack Dev Server</h3>

<p>This is script initialize the Webpack dev server. It loads both the Manager UI and the Storybook preview and starts a HTTP server in a given port.</p>

<p>Source code for Webpack Dev Server is located at: <code>src/index.js</code>.</p>

<h3><a id="user-content-4-static-builder" class="anchor" href="#4-static-builder" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>4. Static Builder</h3>

<p>This is the static builder which creates a set of files which contains your Storybook. Then you can deploy this into GitHub pages or another static files hosting server (or service).
code</p>

<h2><a id="user-content-api" class="anchor" href="#api" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>API</h2>

<p>This is the core set of features of the Storybook. It's expose as a Mantra module with the Redux reducer. This exposes a set of few functionalities to interact with the manager. That includes both the user and the Storybook Preview.</p>

<p>Here are those actions:</p>

<ul>
<li>setStories(stories)</li>
<li>selectStory(kind, story)</li>
<li>addAction(action)</li>
<li>clearActions()</li>
</ul>

<p>For more information about the API, have a look at : <code>src/client/manager/modules/api/actions</code>.</p>

<p>States of these actions can be access using the <code>api</code> field in the reduxStore. It's managed using the <code>api</code> reducer in the API module. You can refer it for more information: <code>src/client/manager/modules/api/configs/reducers</code>.</p>

<h2><a id="user-content-manager---preview-communication" class="anchor" href="#manager---preview-communication" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Manager - Preview Communication</h2>

<p>Manager and Preview communication done using the <code>preview</code> module in the manager. It simply loads the iframe create a communication layer with the Storybook Preview using the page-bus module.</p>

<p>Then, it listen to the events receives from the Storybook Preview and call relevant actions in the <code>api</code> module.
It also watch for the changes in the <code>api</code> field in the reduxStore and emit events to the Storybook Preview.</p>

<h2><a id="user-content-keyboard-shortcuts" class="anchor" href="#keyboard-shortcuts" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Keyboard Shortcuts</h2>

<p>Keyboard Shortcuts are also implemented as a separate module called <code>shortcuts</code>. But, it only manages the reduxStore only.</p>

<p>Keyboard events are defined in <code>src/clibs/key_events.js</code> module. That's because it'll be used in the Manager UI and in the Storybook Preview. We need to track these events in the iframe and send them to the manager to implement the actions.</p>

<p>So, <code>ui</code> and <code>preview</code> modules in the Manager UI call actions in the <code>shortcuts</code> module when there's a Keyboard event.</p>

<p>Then <code>ui</code> module watches reduxStore for those changes apply the shortcut functionality. Refer <code>src/managers/modules/ui/containers/layout.js</code> for some examples.</p>

<h2><a id="user-content-routing" class="anchor" href="#routing" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Routing</h2>

<p>For the app, we don't have a routing system. <code>ui</code> module's <a href="https://kadirahq.github.io/mantra/#sec-Routing-Component-Mounting">Mantra routing hook</a> simply loads the Manager UI to the DOM.</p>

<p>But, we also update the URL for the selected story. That's handles by a config located at <code>src/client/manager/modules/ui/configs/handle_routing.js</code></p>

<h2><a id="user-content-development-setup" class="anchor" href="#development-setup" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Development Setup</h2>

<p>When you are developing the Storybook follow these steps.</p>

<ul>
<li>Run the development watcher with <code>npm run dev</code> inside the local repo.</li>
<li>Link the local repo with <code>npm link</code></li>
<li>Then pick a <a href="https://github.com/kadira-samples/react-storybook-demo">sample app</a> and link storybook into that with <code>npm link @kadira/storybook</code>.</li>
<li>Then run the storybook with <code>DEV_BUILD=1 npm run storybook</code>.</li>
<li><code>DEV_BUILD</code> apply some dev time speed optimizations when developing the manager ui.</li>
<li>Now change code as necessary and see changes.</li>
<li>To reflect changes in the Manager UI, you need to reload the Storybook browser window.</li>
<li>To reflect any changes done in the server code, you need to re-run the storybook command. (<code>DEV_BUILD=1 npm run storybook</code>)</li>
</ul>
